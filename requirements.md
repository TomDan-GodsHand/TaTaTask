# TaTaTask 需求说明书

## 1. 项目概述

TaTaTask 是一个自托管的看板式 TODO 管理 Web 应用。面向家庭/小团队（2-5人），采用 .NET 10 Blazor 技术栈，SQLite 数据存储，用户空间完全隔离。

## 2. 用户管理

### 2.1 注册
- 用户可自由注册，无需邀请码
- 用户名：最长 50 字符，全局唯一，不可为空
- 密码：最少 4 字符，使用 BCrypt 哈希存储
- 注册成功后自动登录并跳转到任务板

### 2.2 登录
- 用户名 + 密码认证
- 登录失败统一提示"用户名或密码错误"（防用户枚举）
- 认证方式：Cookie（有效期 14 天，HttpOnly, SameSite=Lax）

### 2.3 登出
- 清除认证 Cookie
- 重定向到登录页

## 3. 看板（Kanban）

### 3.1 泳道（Status Columns）
系统预设 5 个泳道，全局统一：

| 序号 | 名称   | 说明           |
|------|--------|----------------|
| 1    | 未开始 | 新建任务默认入口 |
| 2    | 进行中 | 正在执行       |
| 3    | 收敛中 | 接近完成，收尾中 |
| 4    | 已完成 | 完成的任务（可归档） |
| 5    | 冻结   | 暂缓/搁置      |

- 泳道不可删除，不可重排
- 横向滚动，每列定宽 260-320px

看板顶部设有一个**临期提醒条**，横向贯穿所有泳道：
- 展示距截止时间不足阈值的任务，**任务不从原泳道移除**，仅作提醒
- 无临期任务时，提醒条**完全隐藏**
- 排序规则：截止时间升序 → 优先级降序

**临期阈值**：
- 全局默认 **48 小时**；用户可在设置中修改全局默认值
- 每个任务可单独设置 `DueWarningHours`（小时），不设则沿用全局默认

### 3.2 任务（TodoItem）
每个任务包含以下属性：

| 字段      | 类型     | 说明                                         |
|-----------|----------|----------------------------------------------|
| Title     | 文本     | 必填，最长200字符                             |
| Priority  | 整数     | 0=无（不显示），1=!，2=!!，3+=!!!              |
| Tags            | 逗号分隔 | 如 "工作,紧急"，展示为独立徽章                  |
| DueDate         | 可选日期 | 过期=红色，今天=黄色，未来=灰色；格式 MM/dd     |
| DueWarningHours | 可选整数 | 临期提醒阈值（小时），null 则沿用全局默认（48h） |
| Status          | 泳道     | 当前所在泳道                                   |
| SortOrder       | 整数     | 泳道内排序，越大越靠后                        |

**泳道内排序规则**：优先级降序 → SortOrder 升序。

**操作**：
- **新建**：在"未开始"泳道底部输入标题，回车或点击 + 创建
- **编辑**：点击卡片菜单的"编辑"，行内修改标题
- **删除**：点击卡片菜单的"删除"，确认后硬删除（连带子步骤）
- **归档**：在"已完成"泳道的卡片菜单出现"归档"选项
- **拖拽移动**：拖拽卡片到目标泳道，自动变更状态并插入顶部位置

### 3.3 子步骤（TodoStep）
每个任务可有多个子步骤：

| 字段        | 类型     | 说明             |
|-------------|----------|------------------|
| Title       | 文本     | 必填，最长200字符 |
| IsCompleted | 布尔     | 勾选切换完成状态  |
| DueDate     | 可选日期 | 格式 MM/dd       |
| SortOrder   | 整数     | 排序             |

**操作**：
- **展开/折叠**：点击任务标题展开/折叠步骤列表
- **添加步骤**：在最下方输入标题 + 点击 +
- **完成切换**：勾选 checkbox 切换完成状态
- **删除步骤**：点击 x 删除

### 3.4 临期提醒

- 当 `DueDate - now <= DueWarningHours` 时，任务出现在临期提示条中
- 提醒条置于所有泳道上方，卡片附带红色边框与剩余时间文本
- 卡片操作与普通任务一致（可编辑/删除/拖拽），拖动后状态变动时仍保留在提醒条中（只要仍满足临期条件）
- 到期后（DueDate 已过）仍保留在提醒条中，直到手动修改截止时间或移除 DueDate

## 4. 归档

### 4.1 归档操作
- 仅"已完成"泳道的任务可归档
- 归档后从看板隐藏，标记 `IsArchived = true`，记录 `ArchivedAt` 时间

### 4.2 归档浏览（/archive）
- 列出所有已归档任务，按归档时间倒序
- **筛选条件**：
  - 标签（Tag 模糊匹配）
  - 归档日期范围（From / To）
  - 标题关键词搜索
- **恢复**：点击"恢复"按钮取消归档，任务回到看板

## 5. 用户空间隔离

- 每个表（TodoItem, TodoStep）包含 `UserId` 列
- 所有 API 查询必须带 `WHERE UserId = @currentUserId`
- `UserId` 从认证 Cookie 中的 Claim 提取
- 用户之间数据完全隔离，无法互相访问

## 6. 技术约束

| 项目       | 选型                                |
|------------|-------------------------------------|
| 运行时     | .NET 10 preview                     |
| 数据库     | SQLite (EF Core)                    |
| 认证       | ASP.NET Core Cookie Authentication  |
| 密码哈希   | BCrypt                              |
| 前端       | MudBlazor 组件框架 + Razor          |
| 渲染       | InteractiveAuto（看板/归档），SSR（登录/注册） |
| 部署       | 单进程自托管，支持 `linux-x64` 发布  |

## 7. 页面清单

| 路由        | 页面       | 渲染模式          | 认证要求     |
|-------------|------------|-------------------|-------------|
| `/`         | 首页       | InteractiveServer | 否（自动跳转到 `/todos`） |
| `/login`    | 登录页     | SSR               | 否           |
| `/register` | 注册页     | SSR               | 否           |
| `/todos`    | 看板页     | InteractiveAuto   | 是           |
| `/archive`  | 归档浏览页 | InteractiveAuto   | 是           |
